Advanced technique 09

Build a custom character without sculpting.

Kitbash existing heads, weapons, shields, pouches, and trophies into a render-only character. The camera only needs the visible parts to look convincing.

← Advanced OverviewAssemble a kitbash ↓
Purpose

Make the library feel personal.

Use it when

A generic STL is close, but a head swap, weapon, shield, pouch, skull, or equipment update would make it the right character.

Not a sculpting tutorial

This is render kitbashing. It does not need to be watertight or printable. A print-ready model needs more cleanup.

Required context

Object Mode, the Outliner, camera framing, and basic transforms: G, R, and S.

Where it fits: before material painting if you want the new parts included in the same palette, or as a separate render-only experiment.
Step by step

Assemble only what the camera sees.

  1. Save a new render-only file. Do not edit the print-ready STL source.
  2. Import a second STL with File → Import → STL. Repeat for head, weapon, accessory, or shield.
  3. Use the Outliner to rename objects: BODY, HEAD_SWAP, WEAPON, SHIELD.
  4. Move with G, rotate with R, and scale with S. Use axis constraints such as G X when useful.
  5. Align roughly from the active camera, then orbit around to catch obvious intersections. The camera-facing result matters most.
  6. Select related parts and group them in a Collection, or parent them to an Empty if you need to move the kitbash as one unit.

Perspective choice

Stay orthographic for the house style. Try slight perspective only when you want a dramatic weapon or hand-forward composition. Check that it still matches the rest of the campaign art.

Composition tools

Crop with intention.

Dramatic crop

Let a large weapon or shoulder break the frame when it communicates character. Keep the face readable.

Rough alignment

Do not spend an hour making hidden joins perfect. Fix only intersections visible from the camera.

Safe organization

Keep imported parts in a separate collection and save versions such as fighter-kitbash-v01.blend.

Common mistake: treating a render-only kitbash like a printable asset. The two goals need different levels of geometry cleanup.
VTT check

Character identity beats mesh purity.

A different weapon, helmet, or trophy can say more than another hour of texture detail. Render what the player needs to recognize.
