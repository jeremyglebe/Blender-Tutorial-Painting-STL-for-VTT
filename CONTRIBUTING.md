# Contributing to STL → VTT Art Lab

Contributions help keep this practical Blender guide useful. Blender changes often, and a short correction can save another beginner a lot of searching.

## Project philosophy

- Keep the Basic Workflow short, direct, and approachable.
- Put advanced concepts in `docs/advanced/` or quick-reference material in `reference/`.
- Prefer practical steps over long theory.
- Include the Blender version when reporting a UI change or render difference.
- Add screenshots when a panel, menu, or visual result is hard to describe.
- Material recipes should include starting settings and a concrete use case.
- Keep external software optional; Blender remains the core workflow.
- Include VTT readability advice when the change affects rendering or composition.

## Useful contribution types

- Blender UI corrections
- Material recipes
- Screenshots and example renders
- Troubleshooting notes
- Spelling and grammar fixes
- Advanced technique pages
- Blender-version-specific notes
- Accessibility and readability improvements

## Basic Workflow protection rule

Please do not add advanced concepts directly into the Basic Workflow unless they are required to complete the basic process. The beginner page should stay short and non-intimidating.

## Reporting an issue

Use the focused [issue forms](https://github.com/jeremyglebe/Blender-Tutorial-Painting-STL-for-VTT/issues/new/choose) when possible. Include:

- Blender version and operating system
- Page and section affected
- Current instruction
- What changed or was confusing
- New menu path or suggested wording
- Screenshot or example render, if useful

## Opening a pull request

1. Fork the repository and make a focused branch.
2. Keep changes scoped to one correction, recipe, page, screenshot set, or site improvement.
3. Test local links and check the page at a narrow viewport.
4. Include the Blender version used for any UI or material change.
5. Open the pull request and complete the provided checklist.

The pull request template asks contributors to confirm that basic documentation remains simple, advanced content stays separated, steps are practical, and external tools remain optional.
