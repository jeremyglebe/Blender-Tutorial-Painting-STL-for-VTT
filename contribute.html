Project philosophy: this guide is for STL-to-VTT art, not general Blender mastery. Contributions should support that workflow.
Before opening an issue

Include enough context to reproduce it.

  1. Include the Blender version and operating system.
  2. Name the affected page and section.
  3. Copy the current instruction and describe the change or point of confusion.
  4. Add the new menu path, setting name, or suggested wording when known.
  5. Attach a screenshot or small example render when it makes the problem clearer.

Pull requests

Make the change directly.

Fork the repository, make a focused change, and open a pull request. Keep beginner docs short, put advanced material outside the Basic Workflow, and test page links before submitting.

Good PRs

One clear correction, recipe, screenshot set, page, or navigation improvement.

Keep software optional

External tools can be mentioned, but they should not become requirements for the core Blender workflow.

Use the checklist

The repository's pull request template asks for Blender version, readability checks, and basic-workflow protection.

Basic Workflow protection: advanced concepts belong outside the beginner page unless required to complete the basic process. The beginner page should stay short and non-intimidating.
